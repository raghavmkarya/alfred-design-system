# Canonical demo data — Northwind Labs

`demo-data.json` is the single fake-data source for every Alfred mock, product shot,
demo environment, deck, email exemplar, and social frame.

**The rule: no fabricated proof.** Alfred has no public customer metrics yet, so every
demo number must be *visibly fictional and internally consistent* — one company, one
story, the same numbers everywhere. If a surface needs a number this file lacks, add it
here first, keep it consistent (spend sums, deltas match series), then use it.

- Company: **Northwind Labs** — fictional 120-person B2B SaaS, ~$310K/month ad spend.
- Signature numbers (already shipped across the system, kept canon here): blended ROAS
  **4.8x**, the **$18K** search→PMax reallocation, the **$4.8K** zero-conversion Meta
  campaign, **$55.1K/mo** recoverable waste, AI Visibility Score **72**, Q3 pipeline
  **$1.84M of $3.2M** at 62% elapsed.
- Never present these as real customer outcomes in public marketing. Market-level stats
  in marketing materials must carry a real citation (see `guidelines/voice-and-naming.md`).
- Components ship with defaults drawn from this dataset; pages composing them should pass
  values from here rather than inventing new ones.
